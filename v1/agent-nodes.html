What agent nodes add to a trading DAG

A trading strategy is naturally a graph of nodes — but in the classic setup every node is a formula, so the strategy can only act on things you can reduce to arithmetic. An agent node is a box with the same plug shape that runs an LLM inside instead: it can read the messy text, go look things up, and make a judgment. Because it drops in like any other node, you can put judgment anywhere a formula used to be the ceiling. The deterministic pipeline stays fast and auditable; the agent nodes fire rarely, right where you’d otherwise need an analyst — and they leave a receipt.

A DAG — directed acyclic graph — is just a flowchart of boxes joined by arrows, where data flows one way from start to finish and never loops back.

The setup

The classic picture, quickly

A quant strategy is naturally a directed graph — each node takes inputs, transforms them, hands the result downstream:

data→ universe→ filters→ features→ score→ rank→ portfolio→ risk→ execute→ attribute

Load the data, cut it to a tradeable universe, drop the risky names, compute signals (momentum, quality, valuation, volatility), combine them into a score, rank, build a portfolio under position/sector/beta limits, turn targets into orders, measure what worked. It’s a nice model — every node is deterministic, testable on its own, and backtestable.

But look at what each of those boxes actually is: a formula. Momentum is a division, a filter is a threshold, the optimizer is a solver. Numbers in, numbers out. If you can’t write the rule as math, the node can’t exist.

That’s fine until you want to trade on the stuff that isn’t numbers:

  • An 8-K dropped — dilutive financing (bad) or an acquisition (good)?
  • The FDA sent a letter — approval, or a rejection dressed up in jargon?
  • “Company X in talks” — the X we hold, or a same-named private firm?
  • The book’s full and a better idea shows up — which holding do we drop?

Those are judgment calls over messy, mostly-text evidence — the stuff a human analyst handles and a spreadsheet can’t. A pure-formula DAG either ignores them or fakes it with a crude proxy (count the scary keywords, cut the biggest loser). That’s the ceiling, and it’s where agent nodes come in.

The idea

A box that can think, where a formula used to sit

An agent node has the same plug shape as any other node — typed inputs, typed outputs, drops into the graph like any box — but instead of a formula, its insides are an LLM that can read, look things up, reason, and hand back a decision.

Formula node

inputs→ a formula→

… a number.

Agent node

inputs→ read · research · judge→

… a decision + confidence + why.

That single substitution — a node that can think, sitting anywhere a formula used to sit — is where the rest of it comes from.

What it buys you

Four things a formula can’t do

  • Text becomes signal

    A formula only eats numbers, so filings, news bodies, earnings-call commentary, and FDA letters are dead weight (at best you count keywords and call it sentiment). An agent reads the actual 8-K and reasons over it — the whole qualitative side of the market turns into a real input. New feeds slot in free, too: a crowd-sentiment API is both a plain “buzz” number and something an agent can weigh as evidence.

  • Judgment where a formula would quietly lie

    “Is this catalyst material?” has a number attached (price moved 4%) but the honest answer needs context a threshold can’t see. Our portfolio_displacer ranks a new idea against the weakest holdings by whether each thesis still has room to run — keeping a red position whose story is alive over a green one whose catalyst is spent. No ranking formula gets that right, because “is the reason still true” isn’t in the price.

  • Research mid-decision, with any tools you hand it

    A formula only sees the inputs you wired in; an agent gets a toolbox — resolve an entity, pull related news, check SEC filings and halts, fetch a URL, and, yes, a plain open web search like you’d run yourself — and it decides on the fly which to reach for and when it has enough. In testing, the displacement node caught a fake catalyst: the claimed contract “did not surface in search results,” so it passed. A formula can’t be skeptical.

  • The universe can come to you

    The classic DAG starts from “all US equities” and filters down on a clock. Agents let you go the other way: a news row or filing lands, wakes a strategy, and spins up a one-shot look at that one name — a universe of one, born from an event, that expires when the event resolves. You can’t build that from formulas, because the interesting part (“is this worth acting on?”) is the judgment.

Depth behind one box

One box, a whole team inside

An agent node is one box on the graph, but that box is just a boundary — what lives inside can be as involved as you want, and the rest of the graph doesn’t know or care. Inside a single “review this catalyst” box, an agent can:

  • go find documents and read them;
  • loop — search → read → “not enough” → search again;
  • spin up sub-agents (one gathers, one tries to refute it, one reconciles);
  • hand off to a cheaper model just to format the JSON;
  • …then emit one decision + confidence + evidence, exactly like the simplest formula node.

That’s real in our stack: portfolio_displacer pulls news per holding, runs a web search per name, reasons across them, and hands off to structure the output — several model calls and downloads inside one box the graph treats as atomic. It scales up, too: the TradingAgents pattern folds a whole firm (analysts → bull/bear debate → trader → risk committee → PM) into one “debate this name” node that outputs buy/pass + confidence + transcript. Two nice consequences:

  • The graph stays simple no matter how smart the nodes get — complexity hides inside the boxes, the wiring between them stays readable.
  • The contract keeps it honest — however elaborate the insides, the node still declares its tools, cost ceiling, timeout, and output shape, and logs what it did. Power on the inside, accountability at the edge. (Debates run a capped number of rounds — an unbounded one is just a runaway bill.)

The space it opens

Why it feels combinatorial

Because an agent node plugs in like any other, you can drop judgment anywhere in the pipeline — not a rewrite, a swap. Each classic box can get a smarter twin sitting next to it, taking just the handful of calls the formula gets wrong:

Classic nodeAgentic version
universe — cap/volume cutoffs “Cleanly tradeable, or a reverse-merger shell?”
feature — count news mentions Read the articles; material positive / neutral / negative, and why
alpha_model — weighted score A catalyst-review step that can veto or bless a name
risk_model — vol/beta “Any pending binary event (FDA, litigation, deal) the vol can’t see?”
optimizer — “book is full” portfolio_displacer — which to keep, judged on live theses
feedback — tweak params “Is the reason we still hold this actually still true?”

Plus a few node types that had no formula version at all — entity resolution, bull/bear debate, self-critique, weighing disagreeing sources instead of averaging them, thesis tracking, agentic exits (deciding to sell is a judgment too). Ten-ish classic node types, each with a judgment upgrade, all composable — that’s the space you were sensing, and it doesn’t really have a fixed size.

Disciplined, not reckless

“So it’s just letting an LLM trade?” — no

That’s the reason to put judgment in a typed graph instead of a chat window:

  • It fires rarely, never every bar

    An agent only runs when something upstream trips it — a news row lands, a condition crosses, the book fills. A handful of calls a day, right where they matter; the fast formulas run everywhere else. (The displacement call costs about $0.04 and only fires when a real winner knocks on a full book.)

  • Everything’s recorded

    Each decision saves what the agent saw, the tools it called, its output, and its reasoning — a receipt you can audit, not a black box.

  • The agent never pulls the trigger

    It emits a decision + confidence; the usual deterministic risk nodes and (optionally) a human gate sit between it and any order.

  • You can check whether the judgment’s any good

    It all runs on paper first — hit-rate by confidence bucket, latency, cost, false positives — and you can A/B two prompts or two models and keep the winner. Live money is gated behind a paper track record.

New arenas

Where this could go: options, crypto, prediction markets

Everything above is stock-picking, but the useful part — judgment over messy evidence — travels to any market where the edge isn’t purely arithmetic. Three where agent nodes open doors a traditional algo can’t:

Options

A classic options algo is all formula — the greeks, the vol surface, delta-hedging. It prices implied volatility to the penny, but can’t judge why it’s there — and options are bets on events with a clock, where the whole question (is the implied move too rich or too cheap?) lives in text.

What an agent node adds

  • Read the actual catalyst and judge direction — calls, puts, or a straddle — instead of trading off IV rank alone.
  • Decide buy vs sell premium: is this binary event likely to be a big move, or a nothing print the market is overpaying for?
  • Map a thesis onto the right structure (“bounded pop → call spread, not naked calls”) and hand the greeks and sizing back to the deterministic layer.
  • Interpret unusual flow: is that big block a hedge, or a directional bet?

Crypto

Crypto trades 24/7 and runs on narrative and headlines — the messy-text world turned up to eleven. Traditional algos arb funding rates and make markets, all on numbers; what they can’t do is read the story that moves the number.

What an agent node adds

  • Judge a depeg in real time: is a stablecoin under $1 a temporary wobble or a real solvency event? (The USDC–SVB weekend is the textbook case — a formula sees “cheap dollar, arb it,” a solvency read says “get out.”)
  • Read token-unlock schedules and tokenomics docs to judge dump risk before a cliff.
  • Tell a genuine protocol exploit from FUD; react to a listing, delisting, or regulatory headline the moment it lands.
  • Verify the contract is the real token, not a same-ticker scam clone — entity resolution and source-checking matter even more here.
  • 24/7 makes the event-born push model native: news breaks at 3am, an agent wakes, investigates one name, and expires.

Kalshi / Polymarket — prediction markets

The purest case. A prediction market is a bet on a plain-English outcome — “Will X happen by date Y?” — often with no price signal at all; fair value is a probability you estimate from evidence. Reading and reasoning is the entire edge.

What an agent node adds

  • Read the resolution criteria closely — these markets live and die on the fine print (“resolves YES only if source Z reports…”) — and catch when the crowd is misreading it.
  • Gather live evidence via web/X search, estimate a probability, compare to the market price, and take the mispriced side.
  • Cross-reference primary sources the instant they drop — a court docket, a data release, an election call — and update faster than the market.
  • Traditional algos can basically only arb the same contract across the two venues; forming the probability from a news event is off-limits to them, because that’s a judgment.

The discipline from the last section still holds everywhere: the agent proposes a decision, confidence, and evidence; deterministic sizing and (optionally) a human gate stand between it and any order; and it earns live capital only after a paper track record.

So it’s not a robot that trades. It’s the same quant pipeline you already trust, that can call an analyst — sparingly, on the record, right where the math runs out.
